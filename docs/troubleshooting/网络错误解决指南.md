# Android ETour 应用网络错误解决指南

## 问题描述
点击登录时显示"网络错误"

## 原因分析
1. **后端服务器未运行**：Spring Boot应用未启动
2. **网络配置错误**：Android设备无法连接到本地服务器
3. **防火墙/端口问题**：端口8081被防火墙阻止
4. **IP地址配置错误**：使用了错误的IP地址

## 解决步骤

### 步骤1：验证后端服务器
1. 打开终端/命令提示符
2. 运行以下命令测试后端：
   ```bash
   # 测试登录接口
   curl -X POST http://localhost:8081/api/user/login -H "Content-Type: application/json" -d '{"username":"admin","password":"123456"}'
   ```

**预期结果**：返回200状态码和JWT令牌

### 步骤2：检查Android网络配置

#### 2.1 确认网络配置
- **开发环境**：使用 `http://10.0.2.2:8081/api/`
- **真机调试**：使用 `http://[你的电脑IP]:8081/api/`

#### 2.2 检查配置文件
文件：`app/src/main/java/com/tour/app/network/ApiClient.java`
```java
private static final String BASE_URL = "http://10.0.2.2:8081/api/";
```

#### 2.3 检查网络权限
文件：`app/src/main/AndroidManifest.xml`
```xml
<uses-permission android:name="android.permission.INTERNET" />
android:usesCleartextTraffic="true"
```

#### 2.4 检查网络安全配置
文件：`app/src/main/res/xml/network_security_config.xml`
```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="true" />
</network-security-config>
```

### 步骤3：根据设备类型配置

#### 3.1 Android模拟器
- 使用IP：`10.0.2.2`
- 无需额外配置

#### 3.2 真机调试
1. 查找电脑IP地址：
   ```bash
   # Windows
   ipconfig
   # 找到IPv4地址，如：192.168.1.100
   ```

2. 修改BASE_URL：
   ```java
   private static final String BASE_URL = "http://192.168.1.100:8081/api/";
   ```

3. 确保手机和电脑在同一WiFi网络

### 步骤4：检查防火墙设置

#### Windows防火墙
1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 添加入站规则：
   - 端口：8081
   - 协议：TCP
   - 允许连接

#### 或者临时关闭防火墙测试：
```bash
# 管理员权限运行
netsh advfirewall set allprofiles state off
```

### 步骤5：使用测试工具

#### 5.1 使用内置测试Activity
运行TestConnectionActivity：
1. 修改AndroidManifest.xml，设置TestConnectionActivity为主Activity
2. 运行应用，测试网络连接

#### 5.2 浏览器测试
打开浏览器访问：
- 开发机：`http://localhost:8081/api/doc.html`
- 手机：`http://[电脑IP]:8081/api/doc.html`

### 步骤6：常见问题解决

#### 6.1 端口被占用
```bash
# 检查端口占用
netstat -ano | findstr 8081
# 杀掉占用进程
taskkill /PID [进程ID] /F
```

#### 6.2 后端启动失败
```bash
# 重新启动后端
cd d:\tour
mvn spring-boot:run
```

## 成功标志
1. 后端服务器正常运行
2. 浏览器能访问API文档
3. curl命令能成功登录
4. Android应用能正常登录

## 故障排除
如果以上步骤仍无法解决：
1. 检查Android Studio的Logcat日志
2. 查看后端控制台是否有请求日志
3. 尝试重启Android Studio和模拟器
4. 检查网络连接是否稳定
